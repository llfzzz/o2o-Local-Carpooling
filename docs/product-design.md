# 产品设计

## 视觉方向

首期前端采用「运营级清爽实用」方向：用户端地图和路线优先，后台以表格、筛选、审核流和状态可视化为核心。

备选方案如下：

1. 地图行动型：H5 第一屏直接进入路线搜索和附近行程，适合高频出行。
2. 信任审核型：突出司机证件、车辆、审核状态和行程安全，适合冷启动建立信任。
3. 运营控制台型：后台强调高密度列表、异常状态、人工复核和审计回溯。

当前实现把 1 和 3 作为 MVP 主目标，2 通过司机认证卡片和后台审核表承接。

## 用户端

- 第一屏展示路线搜索、地图占位区域、可订座行程和订单动作。
- 司机认证流程展示驾驶证、行驶证、OCR Mock、审核状态。
- 所有关键状态使用明确标签：待审核、审核失败、已发布、待支付、已锁座、支付成功、超时取消。
- H5 不保存权威价格、库存或路线距离，界面只展示服务端返回快照。

## 运营后台

- 首页是高密度业务态势：待审核司机、待支付订单、今日行程、风控事件。
- 司机审核表必须同时显示文件、OCR 结果、人工结论和审计入口。
- 订单监控表必须显示订单状态、剩余支付时间、库存占用和取消原因。
- 后台操作默认需要审计：操作者、动作、对象、来源 IP、traceId、时间。
